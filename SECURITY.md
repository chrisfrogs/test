# Security Audit Report

**Date:** 2026-04-14
**Auditor:** Devin (automated security scan)
**Scope:** Full repository contents of `chrisfrogs/test`

---

## Summary

The repository previously contained a single file (`test`) — a **heavily obfuscated Lua script** generated by the [wearedevs.net obfuscator (v1.0.0)](https://wearedevs.net/obfuscator). Analysis determined it to be a **Roblox exploit script** designed to run inside an exploit executor. The file has been removed due to the critical security risks described below.

---

## Critical Findings

### 1. Obfuscated Exploit Code

The script used a VM-based obfuscation pipeline:

1. All 1,852 string literals were encoded with octal escape sequences
2. Strings were further encoded with a **custom base64 alphabet** (non-standard character-to-value mapping)
3. The runtime bytecode interpreter decodes strings on the fly, preventing static analysis

This level of obfuscation is a strong indicator of malicious or policy-violating intent.

### 2. Dangerous Exploit-Only API Usage

The following exploit-executor functions were identified in the decoded string table:

| Function | Category | Risk |
|---|---|---|
| `readfile` | File I/O | Read arbitrary files from the host machine |
| `writefile` | File I/O | Write arbitrary files to the host machine |
| `delfile` | File I/O | Delete arbitrary files on the host machine |
| `isfile` | File I/O | Probe for file existence on the host machine |
| `request` / `http_request` | Network | Send HTTP requests — potential data exfiltration |
| `gethwid` | Fingerprinting | Collect hardware identifiers for tracking |
| `setclipboard` | Clipboard | Modify the system clipboard (hijacking risk) |
| `setfpscap` | Exploit | Manipulate FPS cap (exploit-only function) |
| `getfenv` | Sandbox escape | Access/modify Lua function environments |

### 3. Network Communication

The presence of `request`, `http_request`, `JSONDecode`, `JSONEncode`, and `http` (via `game:GetService`) confirms the script communicates with external servers. The encoded string table likely contains server URLs, API keys, or webhook endpoints that are only revealed at runtime.

### 4. Hardware Fingerprinting

`gethwid` collects a unique hardware identifier, which can be used to track or fingerprint users across sessions.

### 5. Anti-Tamper Protections

The decoded string `"Tamper Detected!"` confirms the script includes anti-analysis mechanisms designed to detect and prevent modification or inspection.

### 6. Roblox Game Manipulation

The script interacts extensively with the Roblox engine via:

- `game:GetService`, `workspace`, `Instance`
- `FireServer` (RemoteEvent exploitation)
- `FindFirstChild`, `GetDescendants`, `GetChildren`, `WaitForChild`
- `Clone`, `Destroy`, `PivotTo`
- `Connect`, `Disconnect`
- `CaptureController`, `ClickButton2`
